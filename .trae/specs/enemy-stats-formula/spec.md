# 敌人属性系统重构 Spec

## Why

需要按照崩坏星穹铁道的数值体系重新设计敌人的属性和血量计算公式。

## What Changes

### 1. 怪物基本设定

* 怪物默认不带有 element（无元素属性），取消怪物的element设计

* 怪物默认 level = 90

### 2. 敌人血量计算公式

**公式**：

```
敌人最终生命值 = 标度系数 × 线性值 × 变体系数 × 精英组系数 × 阶段系数
```

**血量标准（heart）**：

* 标度系数 1.0 时，怪物等级 90 级，血量 = 21997.729

* 定义为 1 heart

**线性值**（通过 index 区分难度）：

* 默认（index=0）：21997.729（1 heart）

* 混沌（index=1）：26954.786

**变体系数**：

* 包括：生命、攻击、速度、韧性、防御系数

* 默认均为 1.0

**精英组系数**：

* 默认 1.0

**阶段系数**：

* 包括：生命、韧性

* 默认 1.0（用于转阶段属性变化）

**生成怪物时的用法**：

* 直接以 n × heart 定义血量

* 例如：10 × heart = 219977.29 血量

### 3. 敌人默认属性（90级普通怪物）

| 属性            | 值                      |
| ------------- | ---------------------- |
| level         | 90                     |
| HP            | 10 × heart = 219977.29 |
| ATK           | 663                    |
| DEF           | 1100                   |
| SPD           | 132.00                 |
| 效果命中          | 32.0%                  |
| 效果抵抗          | 30.0%                  |
| 韧性（toughness） | 40                     |

### 4. 元素抗性规则

* 弱点元素：抗性为 0%

* 非弱点元素：抗性为 20%

**示例**：

* 弱点：物理、虚数、火

* 则：物理、虚数、火 抗性 = 0%

* 其他（雷、冰、风、量子）：抗性 = 20%

## Impact

* Affected code:

  * `src/game/models.py` - Character 模型修改

  * `src/game/battle.py` - create\_enemy 函数

  * `data/enemies.json` - 敌人数据模板

## ADDED Requirements

### Requirement: 敌人血量公式

系统 SHALL 实现敌人血量计算公式，支持多种难度和系数。

### Requirement: 元素抗性计算

系统 SHALL 根据弱点元素计算敌人的元素抗性。

## MODIFIED Requirements

### Requirement: create\_enemy 函数

修改 `create_enemy` 函数，使用新的默认属性值。

### Requirement: Character 模型

修改 Character 模型支持新的敌人属性计算。
