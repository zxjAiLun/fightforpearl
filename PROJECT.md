# Fight for Pearl — 项目进度总结 v0.3.0

## 项目概述

**Fight for Pearl** 是一个回合制战斗框架，灵感来源于崩坏星穹铁道的战斗系统。当前版本 **v0.3.0**，核心战斗逻辑已基本完成。

## 项目目标

**复刻崩坏星穹铁道回合制战斗逻辑**
- 优先级：战斗逻辑 > GUI > 养成系统（暂不实现）
- 养成逻辑目前用手填数据

---

## 已完成功能

### ✅ 战斗引擎核心
- [x] 回合制战斗流程
- [x] 行动值（Action Value）系统 — 基于速度的行动排序
- [x] 回合标记（Round Marker）机制
- [x] Step Back功能（回退到上一个行动）
- [x] DOT触发和结算
- [x] 行动角色选择bug修复（max→min）

### ✅ 技能系统
- [x] 普通攻击（Basic Attack）— 不消耗战绩点，回复1点战绩点
- [x] 战技（Special Skill）— 消耗1点战绩点，回复能量
- [x] 大招（Ultimate）— 消耗全部能量，需能量满
- [x] 追击（Follow-Up）— 满足条件后触发的额外攻击
- [x] 被动技能触发系统
- [x] AOE技能支持

### ✅ 能量与战绩点系统
- [x] 能量上限：120点
- [x] 初始能量：60点
- [x] 普攻回复：20能量
- [x] 战技回复：30能量
- [x] 战绩点：团队共享，上限6点
- [x] **战技消耗1点战绩点**

### ✅ 效果命中/抵抗系统
- [x] 命中率公式：`效果命中 / (效果命中 + 效果抵抗 + 23)`
- [x] DOT效果需要判定命中率
- [x] 控制效果（冻结/纠缠/禁锢）需要判定命中率
- [x] 裂伤（HP%伤害）无需判定，始终命中

### ✅ 敌人AI
- [x] 优先使用大招（能量满时）
- [x] 有多个目标时优先使用AOE技能
- [x] 优先攻击血量最低的目标
- [x] 敌人AI配置文件（enemy_ai.json）

### ✅ 伤害计算
- [x] 崩铁风格伤害公式
- [x] 防御减伤公式（两种）
- [x] 暴击判定（敌人不暴击）
- [x] 易伤区叠加
- [x] 增伤区叠加
- [x] 击破伤害计算

### ✅ 击破系统
- [x] 7种元素对应7种击破效果
- [x] 裂伤、灼烧、冻结、触电、风化、纠缠、禁锢

### ✅ GUI界面
- [x] Pygame图形界面

## 最近更新 (v0.3.0)

### 核心修复
1. **战技消耗战绩点** — 战技执行前检查战绩点是否足够
2. **行动值常量统一** — 统一为AV_BASE=10000
3. **行动角色选择bug** — max改为min
4. **FollowUpRule.check_trigger方法** — 缺失方法已添加
5. **SkillType.FOLLOW_UP** — 枚举值缺失已添加

### 新增功能
1. **效果命中/抵抗系统** — DOT和控制效果需要判定命中率
2. **敌人AI智能选择** — 大招优先、AOE优先、攻击血量最低目标
